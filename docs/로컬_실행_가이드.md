# 로컬 실행 가이드

## 사전 준비

- **Flutter 3.x** 이상 설치 ([flutter.dev](https://flutter.dev/docs/get-started/install))
- **Python 3** (로컬 웹 서버 실행용 — macOS 기본 내장)
- **Chrome** 또는 **Safari** 브라우저

---

## 1. 인덱스 페이지 (보드게임 목록) 로컬 실행

인덱스 페이지(`index.html`)는 순수 HTML/CSS/JS로 구성되어 있어 별도 빌드가 필요없습니다.  
단, 게임 카드의 링크(`games/skulking/`, `games/yacht/`)가 Flutter 빌드 결과물을 참조하므로, **로컬 HTTP 서버**로 실행해야 합니다.

### 실행 방법

```bash
# 프로젝트 루트로 이동
cd /path/to/quick-board

# Python 내장 HTTP 서버 실행 (포트 8000)
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000` 접속.

> `open http://localhost:8000` 명령으로 브라우저를 바로 열 수 있습니다.

### 주의사항

- `file://` 프로토콜로 `index.html`을 직접 열면 게임 카드 링크가 정상 동작하지 않습니다.
- 게임 카드 클릭 시 Flutter 앱이 로드되려면 `games/skulking/`, `games/yacht/` 디렉토리에 Flutter 빌드 결과물이 있어야 합니다. ([Flutter 앱 빌드 방법](#2-flutter-게임-앱-빌드--실행) 참고)

## 2. Flutter 게임 앱 빌드 & 실행

Flutter 앱은 **웹 빌드 후 정적 파일을 `games/` 디렉토리에 복사**하는 방식으로 인덱스 페이지와 통합됩니다.

### 디렉토리 구조

```
quick-board/
├── index.html                          # 메인 인덱스 페이지
├── games/
│   ├── skulking/                       # 스컬킹 Flutter 웹 빌드 결과물
│   └── yacht/                          # 요트다이스 Flutter 웹 빌드 결과물
└── quick-board-flutter/
    ├── apps/
    │   ├── skulking/                   # 스컬킹 Flutter 소스
    │   └── yacht/                      # 요트다이스 Flutter 소스
    └── packages/
        └── quick_board_core/           # 공통 패키지
```

### 의존성 설치

처음 클론하거나 `pubspec.yaml`이 변경된 경우에 실행합니다.

```bash
# 스컬킹
cd quick-board-flutter/apps/skulking
flutter pub get

# 요트다이스
cd quick-board-flutter/apps/yacht
flutter pub get
```

### Flutter 웹 빌드

```bash
# 스컬킹 빌드
cd quick-board-flutter/apps/skulking
flutter build web --base-href /quick_board/games/skulking/

# 요트다이스 빌드
cd quick-board-flutter/apps/yacht
flutter build web --base-href /quick_board/games/yacht/
```

### 빌드 결과물 복사

```bash
# 프로젝트 루트로 이동
cd /path/to/quick-board

# 스컬킹
rm -rf games/skulking
cp -r quick-board-flutter/apps/skulking/build/web games/skulking

# 요트다이스
rm -rf games/yacht
cp -r quick-board-flutter/apps/yacht/build/web games/yacht
```

### 실행 확인

```bash
# 프로젝트 루트에서 HTTP 서버 실행
python3 -m http.server 8000
```

브라우저에서 아래 URL로 직접 접속해 확인할 수 있습니다.

| 페이지 | URL |
|--------|-----|
| 인덱스 (게임 목록) | `http://localhost:8000` |
| 스컬킹 | `http://localhost:8000/games/skulking/` |
| 요트다이스 | `http://localhost:8000/games/yacht/` |

## 3. 한 번에 빌드 & 실행 (스크립트)

아래 명령을 프로젝트 루트에서 실행하면 두 앱을 모두 빌드하고 서버를 시작합니다.

```bash
cd /path/to/quick-board

# 스컬킹 빌드 & 복사
(cd quick-board-flutter/apps/skulking && flutter build web --base-href /quick_board/games/skulking/) && \
rm -rf games/skulking && cp -r quick-board-flutter/apps/skulking/build/web games/skulking

# 요트다이스 빌드 & 복사
(cd quick-board-flutter/apps/yacht && flutter build web --base-href /quick_board/games/yacht/) && \
rm -rf games/yacht && cp -r quick-board-flutter/apps/yacht/build/web games/yacht

# 서버 실행
python3 -m http.server 8000
```

## 4. Flutter 앱 개발 모드 실행 (Hot Reload)

빌드 없이 빠르게 UI를 확인할 때는 각 앱을 Chrome에서 직접 실행합니다.  
이 경우 인덱스 페이지와 연동은 안 되지만, 개발 중 Hot Reload가 동작합니다.

```bash
# 스컬킹
cd quick-board-flutter/apps/skulking
flutter run -d chrome

# 요트다이스
cd quick-board-flutter/apps/yacht
flutter run -d chrome
```

> Hot Reload: 코드 변경 후 터미널에서 `r` 입력  
> Hot Restart: `R` 입력  
> 종료: `q` 입력

## 5. 모바일 앱 실행 (iOS / Android)

```bash
# 연결된 기기 확인
flutter devices

# 스컬킹 — iOS 시뮬레이터
cd quick-board-flutter/apps/skulking
flutter run -d <device-id>

# 요트다이스 — iOS 시뮬레이터
cd quick-board-flutter/apps/yacht
flutter run -d <device-id>
```

> Android 실행 시 `JAVA_HOME` 환경 변수와 Android SDK 설정이 필요합니다.
