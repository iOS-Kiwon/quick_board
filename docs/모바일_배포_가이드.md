# 모바일 배포 가이드 (iOS / Android)

## 사전 준비

### 공통
- Flutter SDK 설치 및 `flutter doctor` 통과
- 프로젝트 루트: `quick-board-flutter/apps/skulking/`

### iOS
- macOS + Xcode 설치
- Apple ID (무료) 또는 Apple Developer Program 계정 ($99/년)

### Android
- Android Studio 또는 JDK 17+ 설치
- Google Play Console 계정 ($25 일회성) — Play Store 배포 시 필요

---

## iOS 배포

### 배포 방식 비교

| 방식 | 필요 계정 | 용도 |
|---|---|---|
| 내 기기 직접 설치 | 일반 Apple ID | 개인 테스트 (최대 3대) |
| TestFlight | Developer Program | 베타 배포 (최대 10,000명) |
| App Store | Developer Program | 공개 출시 |

### 1단계 — Flutter 빌드

```bash
cd quick-board-flutter/apps/skulking
flutter build ios --release
```

### 2단계 — Xcode에서 열기

```bash
open ios/Runner.xcworkspace
```

> `Runner.xcodeproj`가 아닌 `.xcworkspace`를 열어야 합니다.

### 3단계 — Xcode 서명 설정

1. 왼쪽 트리에서 `Runner` 선택
2. `Signing & Capabilities` 탭
3. `Team` → Apple ID로 로그인 후 팀 선택
4. `Bundle Identifier` 설정 (예: `com.yourname.skulking`)
5. 상단 기기 선택: 실기기 또는 `Any iOS Device (arm64)`

### 4단계 — Archive

```
Xcode 메뉴 → Product → Archive
```

빌드 완료 후 **Organizer** 창이 자동으로 열립니다.

### 5단계 — 배포

Organizer에서 `Distribute App` 클릭 후 목적에 따라 선택:

| 목적 | 선택 항목 |
|---|---|
| 내 기기 직접 설치 | Development |
| TestFlight / App Store | App Store Connect |

App Store Connect 선택 시 → [appstoreconnect.apple.com](https://appstoreconnect.apple.com) 에서 심사 제출

### 주의사항

- `Info.plist`에 AdMob App ID가 설정되어 있어야 빌드 통과
  - 현재 값: `ca-app-pub-5980133283002959~1048900622` ✅
- App Store 출시 시 개인정보처리방침 URL 필수

## Android 배포

### 배포 방식 비교

| 방식 | 필요 계정 | 용도 |
|---|---|---|
| APK 직접 설치 | 없음 | 개인 테스트 |
| 내부 테스트 트랙 | Play Console | 소규모 테스트 (최대 100명) |
| 공개 테스트 / 출시 | Play Console | 베타 / 정식 출시 |

### 1단계 — 서명 키 생성 (최초 1회)

```bash
keytool -genkey -v -keystore ~/skulking-release.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias skulking
```

> 생성한 `.jks` 파일은 분실하면 재배포 불가. 안전한 곳에 백업 필수.

### 2단계 — 서명 설정

`quick-board-flutter/apps/skulking/android/key.properties` 파일 생성:

```properties
storePassword=<키스토어 비밀번호>
keyPassword=<키 비밀번호>
keyAlias=skulking
storeFile=<키스토어 절대 경로>/skulking-release.jks
```

`android/app/build.gradle`에 서명 설정 추가:

```groovy
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

android {
    ...
    signingConfigs {
        release {
            keyAlias keystoreProperties['keyAlias']
            keyPassword keystoreProperties['keyPassword']
            storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
            storePassword keystoreProperties['storePassword']
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
        }
    }
}
```

### 3단계 — Flutter 빌드

```bash
cd quick-board-flutter/apps/skulking

# App Bundle (Play Store 권장)
flutter build appbundle --release

# APK (직접 설치용)
flutter build apk --release
```

빌드 결과물 위치:
- App Bundle: `build/app/outputs/bundle/release/app-release.aab`
- APK: `build/app/outputs/flutter-apk/app-release.apk`

### 4단계 — APK 직접 설치 (테스트용)

```bash
flutter install
# 또는
adb install build/app/outputs/flutter-apk/app-release.apk
```

기기에서 `설정 → 알 수 없는 앱 설치` 허용 필요.

### 5단계 — Play Store 배포

1. [play.google.com/console](https://play.google.com/console) 접속
2. 앱 만들기 → 앱 이름, 기본 언어, 앱/게임 선택
3. `프로덕션` 또는 `내부 테스트` 트랙 선택
4. `app-release.aab` 업로드
5. 스토어 등록 정보 입력 (설명, 스크린샷, 아이콘)
6. 심사 제출

### 주의사항

- `AndroidManifest.xml`에 AdMob App ID가 설정되어 있어야 함
  - 현재 값: `ca-app-pub-5980133283002959~7000008878` ✅
- `key.properties`는 `.gitignore`에 추가해서 커밋하지 않도록 주의
- 최초 Play Store 심사는 7일 내외 소요
